## 二、Git 三方合并升级法（推荐）

### Step 1：下载新版本 tarball

```bash
VERSION=v1.7.3  # 替换为目标版本

curl -fsSL -o /tmp/cordys-${VERSION}.tar.gz \
  "https://api.github.com/repos/1Panel-dev/CordysCRM/tarball/${VERSION}"

mkdir -p /tmp/cordys-${VERSION}-source
tar xzf /tmp/cordys-${VERSION}.tar.gz -C /tmp/cordys-${VERSION}-source --strip-components=1
```

### Step 2：更新 upstream 纯净分支

```bash
git checkout upstream
rsync -a --delete --exclude='.git' --exclude='target' \
  --exclude='node_modules' --exclude='dist' --exclude='.turbo' \
  /tmp/cordys-${VERSION}-source/ ./
git add -A
git commit -m "${VERSION} upstream"
```

### Step 3：三方合并到 master

```bash
git checkout master
git merge upstream -m "merge: upstream ${VERSION} → master"
```

### Step 4：处理冲突

```bash
# 1) 上游删除的文件 → 删除
git status --porcelain | grep "^DU" | awk '{print $2}' | xargs git rm

# 2) 双方都新增的文件 → 取上游版本
git status --porcelain | grep "^AA" | awk '{print $2}' | while read f; do
  git checkout --theirs "$f" && git add "$f"
done

# 3) 双方都修改的文件 → 手动合并
git diff --name-only --diff-filter=U
```

### Step 5：编译验证

```bash
# 后端（必须编译 app 模块，不是 crm 模块！app 才有 spring-boot-maven-plugin）
./mvnw install -N -DskipTests -q
./mvnw install -pl framework,app -DskipTests -DskipAntRunForJenkins -q --file backend/pom.xml
# 产物: backend/app/target/app-main.jar（可执行 fat JAR）

# 前端
cd frontend && pnpm i -w --no-frozen-lockfile
cd packages/web && pnpm build
# 或用容器: rebuild_backend.sh + start_frontend.sh
```

## 四、踩坑经验与解决方案

### 🔴 严重问题

#### 1. Untracked 自定义文件在 git stash 时丢失
- **现象**：`dict-manage/`、`welltrans-push/` 等前端页面切换分支时丢失
- **原因**：`git stash push` 默认不包含 untracked 文件，需要 `git stash push -u`
- **解决**：从备份目录 `~/cordys-crm-dev/CordysCRM-1.7.0/` 恢复
- **预防**：**升级前必须 `git add` 所有自定义文件并 commit**，不允许有 untracked 文件

#### 2. pkill 在容器中卡死
- **现象**：`pkill -f 'crm-main.jar'` 无响应，阻塞脚本
- **原因**：pkill 遍历进程时遇到僵尸进程（defunct）卡住
- **解决**：改用 `ps aux | grep java | grep -v defunct | awk '{print $2}' | xargs kill`

#### 3. 编译产物是 crm-main.jar 不可执行
- **现象**：`java -jar crm-main.jar` → "no main manifest attribute"
- **原因**：`crm` 模块是纯业务 JAR（无 Spring Boot 插件），`app` 模块才打包可执行 fat JAR
- **解决**：编译命令改为 `-pl framework,app`，产物路径 `backend/app/target/app-main.jar`

### 🟡 中等问题

#### 4. 前端 ESLint 错误系列
- **现象**：Vite HMR overlay 报 `simple-import-sort/imports` error
- **原因**：合并的自定义代码未遵循 v1.7.2 项目的 ESLint 规范
- **涉及文件**：`select.vue`, `radio.vue`, `checkbox.vue`, `optionConfig.vue`, `dict-manage/index.vue`, `welltrans-push/index.vue`
- **解决**：`pnpm exec eslint --fix <file>` 自动修复 import 排序 + prettier 格式

#### 5. no-use-before-define 错误
- **现象**：`dict-manage/index.vue` 中 `loadItems()`、`openItemDrawer()`、`delItem()` 使用前未定义
- **原因**：`itemCols` 的 render 函数引用了后面才定义的函数；`selectCat()` 调用了后面才定义的 `loadItems()`
- **解决**：调函数定义顺序，确保所有被引用的函数在其引用者之前定义

#### 6. Import 排序规则（simple-import-sort）
- **分组规则**（每组间空行分隔，组内按字母序）：
  1. 外部包：`lodash-es` < `naive-ui` < `vue` < `vue-draggable-plus`
  2. `@lib/` 别名导入
  3. `@/` 别名导入
  4. 相对路径导入 `../../`
- **注意**：`eslint --fix` 会自动处理，不需要手动排序

#### 7. 前端代理到后端失败
- **现象**：`ECONNREFUSED 127.0.0.1:8081`
- **原因**：Vite 开发服务器在 `cordys-frontend` 容器中，`127.0.0.1:8081` 无法访问 `cordys-backend` 容器
- **解决**：设置环境变量 `VITE_DEV_DOMAIN=http://cordys-backend:8081`

#### 8. frontend 嵌入 .git 仓库
- **现象**：`git add` 时出现 "adding embedded git repository: frontend" 警告
- **原因**：commit `38c8b5b` 整合 frontend 时直接复制了整个目录含 `.git`
- **解决**：`rm -rf frontend/.git`，在 upstream 分支中处理

#### 9. ESLint `simple-import-sort` 配置
- **eslint 配置**：`frontend/packages/web/eslint.config.cjs`
- **插件**：`eslint-plugin-simple-import-sort`
- **修复命令**：`pnpm exec eslint --fix <file>`（在容器中需用 `pnpm exec` 而非 `npx`）

### 🟢 注意事项

#### 10. 部署脚本要点
- 启动前端必须提前 `VITE_DEV_DOMAIN=http://cordys-backend:8081`
- MySQL 和 Redis 容器名：`cordys-mysql`、`cordys-redis`
- 数据库名：`cordys-crm-1.7.0`，用户：`root/root`
- Flyway 配置：`validate-on-migrate=false`，migration 位置 `classpath:migration`
- 管理员账号：`admin` / `CordysCRM`

#### 11. Flyway 迁移注意
- v1.7.0 自定义迁移（公海、Welltrans、字典）已在数据库中执行
- v1.7.2 新增迁移自动执行（V1.7.2_1__init.sql, V1.7.2_2__ga_ddl.sql, V1.7.2_2_1__data.sql）
- 如果 `validate-on-migrate=true`，重复执行的 DML 会报错，需用 `INSERT IGNORE`

## 七、未来升级快速参考

```bash
# 1. 确保所有文件已 commit（禁止 untracked 文件）

  "https://api.github.com/repos/1Panel-dev/CordysCRM/tarball/${VERSION}"

tar xzf /tmp/cordys-${VERSION}.tar.gz -C /tmp/cordys-${VERSION}-source --strip-components=1

# 3. 更新 upstream 分支
git checkout upstream
rsync -a --delete --exclude='.git' --exclude='target' \
  --exclude='node_modules' --exclude='dist' /tmp/cordys-${VERSION}-source/ ./
git add -A && git commit -m "${VERSION} upstream"

# 4. 合并
git checkout master && git merge upstream

# 5. 解决冲突后编译验证
bash rebuild_backend.sh       # 后端编译
bash start_backend.sh         # 启动后端（端口 15173）

# 6. 修复前端 ESLint（如有 error overlay）
docker exec cordys-frontend bash -c "
  cd /tmp/frontend-1.7.2/packages/web && \
  pnpm exec eslint --fix 'src/**/*.vue'
"

# 7. 验证通过后推送
git push origin master upstream
```
